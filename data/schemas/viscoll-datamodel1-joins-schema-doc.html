Documentation and Schema for the "Joins" version of the collation model. The "Joins" version lists conjoins within quires. The "Leaves" version lists leaves, and properties of leaves (including conjoins as attribute values), within quires.

1. Elements

1.1. <leaf/>

<leaf/> Describes a single leaf.
Namespacehttp://schoenberginstitute.org/schema/collation
Modulederived-module-joins-schema
Attributes
modeNotes the mode of the leaf, whether original to the manuscript, added at any point, missing, or replaced. If this leaf is a "false conjoin" for a singleton, @mode will not exist.
StatusOptional
Legal values are:
original
added
missing
replaced
singleIs the leaf a singleton? True or false. If this leaf is a "false conjoin" for a singleton, @mode will not exist.
StatusOptional
Legal values are:
false
[Default]
true
folio_numberThe number of the folio (may also be pages, separated by a dash, e.g. 121-122). If this leaf is a "false conjoin" for a singleton, @mode will not exist.
StatusOptional
Datatype
text
conjoinThe leaf@n of the leaf that is conjoined with this leaf. If there is no conjoin, this will be blank.
StatusRequired
positionPosition of the leaf in the quire. All leaves have position, even false conjoins.
StatusRequired
Datatype
data.numeric
nNumber of the leaf in the physical quire. If the leaf is a false conjoin, @n will be empty.
StatusRequired
Datatype
text
Contained by
derived-module-joins-schema: unit
May containEmpty element
Declaration
element leaf
{
   attribute mode { "original" | "added" | "missing" | "replaced" }?,
   attribute single { "false" | "true" }?,
   attribute folio_number { text }?,
   attribute conjoin { text },
   attribute position { data.numeric },
   attribute n { text },
   empty
}
